# ROBO-ONE Beginners Auto BLE PC Controller

## はじめに
このプロジェクトは、ROBO-ONEの自動機に挑戦する学生のための
BLE通信実験用PCコントローラソフトウェアです。

---

## このプロジェクトについて

私は現在、ROBO-ONE beginners などの自動機に挑戦するためのロボットを製作しています。
しかし開発を進める中で、BLE（Bluetooth Low Energy）の通信を扱う部分で多くのつまずきがありました。

特に、

* スマートフォンのBLEアプリでは操作がしづらい
* 通信のテストやデバッグがやりにくい
* プログラム開発との連携が難しい

といった点がありました。

そこで、BLE通信を **PCから扱いやすくするための簡単なコントローラソフト（Python）** を作成しました。

---

## 公開の目的

このリポジトリでは、そのPC用BLEコントローラソフトを公開しています。

ただし、このソフトは **開発途中のもの**であり、
現在自分が使用している最新のバージョンとは少し異なります。

公開しているものは、

* できるだけシンプル
* 動作の基本が分かる
* 改造の入口になる

ということを目的に、**一番簡単な状態のもの**にしています。

## 概要

本ソフトウェアは、ROBO-ONE 初心者向けロボットの BLE 通信を
**PCから制御するためのコントローラソフトウェア**です。

従来、BLE通信にはスマートフォンアプリケーション（Bacon や Bluetooth Connect など）を使用することができますが、

* 接続の安定性
* 操作インターフェースの制約
* 連続的な操作や実験の難しさ

など、ロボット制御用途では不便な点が多くありました。

そこで本プロジェクトでは、**PCからBLE通信を行いロボットを制御できるソフトウェア**を作成しました。
これにより、ロボットの操作や制御実験をより柔軟に行うことができます。

# 使い方

## 0. pythonをダウンロード
[公式サイト](https://www.python.org/downloads/)から最新版のpython3をダウンロード、展開します。

## 1. 必要なライブラリをインストール

BLE通信に必要なPythonライブラリをインストールします。

```bash
pip install bleak
```

必要に応じて、他の依存ライブラリもインストールしてください。

---

## 2. ロボット側プログラムを起動

ROBO-ONEロボットのマイコンに
BLE対応プログラム（MicroPython）を書き込み、起動します。

ロボットがBLEデバイスとして **advertising（アドバタイズ）状態** になっていることを確認します。

---

### 補足：BLEのUUIDを調べる方法

BLE通信では、データの送受信に **UUID（通信の識別子）** が使われます。
ロボットと通信するためには、このUUIDを確認する必要があります。

UUIDは、BLEスキャンツールを使うことで調べることができます。

例：

* nRF Connect（スマートフォンアプリ）
* BLE Scanner
* Bluetooth Developer Tools（PC）

基本的な手順：

1. BLEスキャンアプリを起動する
2. ロボットの電源を入れる
3. デバイス一覧からロボットを選択する
4. **Service** や **Characteristic** の一覧を表示する
5. 表示される **UUID** を確認する

そのUUIDをPC側プログラムに設定することで通信できます。

※ BLEは最初は少し分かりにくいですが、ツールで実際に確認すると仕組みが理解しやすくなります。
※ 多くの場合 **Nordic UART Service** が使用されています。

---

## 3. PC側ソフトウェアを起動

PC上でコントローラプログラムを実行します。

```bash
python ble_robot.py
```

または

```bash
python main.py
```

（実際のファイル名に合わせてください）

---

## 4. BLE接続

プログラム起動後、ロボットのBLEデバイスを検索して接続します。
接続が成功すると、PCからロボットにコマンドを送信できるようになります。

# 通信仕様（BLEコマンド）

本ソフトウェアはBLE（Bluetooth Low Energy）を用いて
PCとロボット間でコマンド通信を行います。

通信構成：

* PC：BLE Central（接続側）
* Robot：BLE Peripheral（被接続側）

PC側ソフトウェアがロボットへコマンドを送信し、
ロボット側のMicroPythonプログラムがそれを受信して動作します。

通信は主に **文字列コマンド** で行われます。

例：

```
F
B
L
R
X
```

---

# 接続構成

```
+--------------------+
|        PC          |
|  BLE Controller    |
|  (Python Program)  |
+---------+----------+
          |
          | BLE通信
          |
+---------v----------+
|      Robot         |
|  MicroPython BLE   |
|  Servo Controller  |
+--------------------+
```

通信の流れ：

1. ロボットがBLEデバイスとしてadvertisingを開始
2. PCソフトウェアがデバイスを検索
3. PCからBLE接続
4. コマンド送信
5. ロボットが動作

---

# 操作コマンド一覧

| コマンド | 動作      |
| ---- | ------- |
| F    | 前進      |
| B    | 後退      |
| L    | 左回転     |
| R    | 右回転     |
| X    | 停止      |
| A    | 自動制御モード |
| OP   | 手動操作モード |

※ 実際のコマンドはロボット側プログラムの実装に依存します。

---

# 通信例

PC側からロボットへコマンドを送信する例

```
send("F")
```

ロボット側では

```
受信コマンド: F
→ 前進動作
```

のように処理されます。
